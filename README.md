# 🛒 End-to-End Data Pipeline for E-commerce and Sales Analytics

## 📌 Project Overview
This project builds a complete data pipeline using **AWS S3**, **Python (Pandas + Boto3)**, and an **auto-generated HTML Analytics Dashboard**.

Raw e-commerce CSV files are stored in S3, cleaned and transformed using Python ETL, and the processed data is used to auto-generate a business insights dashboard.

---

## 🏗️ Project Architecture

```
customer.csv / product.csv / sales.csv
            ↓
    AWS S3 (Raw Storage)
    Bucket: ecommerce-sales-pipeline-2026/raw-data/
            ↓
    Python ETL Pipeline
    (Pandas — cleaning & transformation)
    (Boto3 — S3 connection)
            ↓
    AWS S3 (Processed Storage)
    Bucket: ecommerce-sales-pipeline-2026/processed-data/
    ├── final_processed_data.csv
    └── dashboard.html
            ↓
    HTML Analytics Dashboard
    (Auto-generated by ETL pipeline)
            ↓
    Business Insights
```

## 🚀 How to Run This Project

### Step 1 — AWS Setup (One time only)
Make sure AWS credentials are configured on your machine:
```bash
aws configure
```
Enter your:
- AWS Access Key ID
- AWS Secret Access Key
- Region: `ap-south-1`

### Step 2 — Install Required Libraries
```bash
pip install pandas boto3
```

### Step 3 — Upload Raw Files to S3
Go to AWS Console → S3 → `ecommerce-sales-pipeline-2026` → `raw-data` folder  
Upload these 3 files:
- `customer.csv`
- `product.csv`
- `sales.csv`

### Step 4 — Run the ETL Pipeline
```bash
cd scripts
python etl_pipeline.py
```

### Step 5 — Expected Terminal Output
```
Connected to AWS S3
All raw files downloaded successfully
Datasets Loaded Successfully
Data cleaned successfully
Data transformation completed
Processed CSV created
Processed file uploaded to S3
Generating Analytics Dashboard...
Dashboard HTML generated successfully
Dashboard uploaded to S3
ETL Pipeline Completed Successfully
```

### Step 6 — View Dashboard
Open this file in any browser:
```
processed_data/dashboard.html
```

## 🔄 How to Use With Different Data

### If only data changes (same columns):
```bash
# Just run the pipeline again — nothing else needed
python etl_pipeline.py
```

### If file names change:
Open `etl_pipeline.py` and update only these 3 lines at the top:
```python
customer_file_key = 'raw-data/YOUR_NEW_CUSTOMER_FILE.csv'
product_file_key  = 'raw-data/YOUR_NEW_PRODUCT_FILE.csv'
sales_file_key    = 'raw-data/YOUR_NEW_SALES_FILE.csv'
```

### If column names change:
Open `etl_pipeline.py` and find the section:
```python
# Detect column names flexibly
sales_col  = next((c for c in df.columns if 'sale'   in c.lower()), None)
profit_col = next((c for c in df.columns if 'profit' in c.lower()), None)
```
Update the keyword inside the quotes to match your new column names.
